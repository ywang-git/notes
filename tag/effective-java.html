Chapter 1. Introduction

Fundamental principles

  • clarity and simplicity
  • no surprises for component's behaviors
  • smaller component, but no smaller
    • component: any reusable software element, method -> packages
  • dependencies between components should be minimum
  • fail fast (error detected as early as possible)

exported API: API elements that are accessible outside of the package that defines the API. public, protected members and constructors of every public class or interface in the package.
